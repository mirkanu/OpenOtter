## Layout Patterns

### Recordings List Page (`/recordings`)
- **Layout:** Single column, mobile-first
- **Spacing:** 16px container padding, 24px between cards
- **Empty state:** Centered icon + heading + body text
- **Card structure:** Recording title (top), metadata row (date, duration, speakers), status badge (right)

### Individual Recording Page (`/recordings/[id]`)
- **Layout:** Audio player (sticky top), transcript (scrollable below)
- **Audio player:** Fixed position on mobile, native controls with speed toggle
- **Transcript:** Stacked utterances with active highlighting
- **Sync behavior:** Current utterance highlighted in indigo-100 background, left border indicator

---

## Interaction Contracts

### Audio Player Sync
**Event:** `timeupdate` fires (4-10 times per second)
**Action:** Find utterance where `currentTime >= start && currentTime <= end`, apply highlight class
**Performance:** Throttle updates to 100ms minimum to prevent re-render lag

### Transcript Click
**Event:** User clicks transcript segment
**Action:** Audio player seeks to `utterance.start / 1000` seconds, auto-plays
**Feedback:** Clicked segment highlights immediately, audio begins within 100ms

### Recording Card Click
**Event:** User clicks recording card
**Action:** Navigate to `/recordings/[id]` via Next.js Link
**Feedback:** Visual active state on press (CSS :active)

### Delete Recording
**Event:** User clicks delete button
**Action:** Show confirmation modal/dialog
**Confirmation:** "Delete Recording: This will permanently delete..."
**On Confirm:** Call DELETE /api/recordings/[id], remove database entry, delete audio file, navigate to list

## Mobile-First Responsive Rules

1. **Touch targets:** Minimum 44px height for all clickable elements (recording cards, buttons)
2. **Audio player:** Full-width on mobile, controls always visible (no hide-on-play)
3. **Transcript:** Single column, full-width utterance blocks, readable line-length
4. **Typography:** Minimum 14px body text, 16px for better readability on mobile
5. **Spacing:** Increased vertical spacing on mobile (24px between sections)

## Implementation Notes

**Database Integration:**
- Recordings list fetches from GET /api/recordings
- Individual recording fetches from GET /api/recordings/[id]
- Transcript stored as JSON in utterances_json column (per RESEARCH.md)

**Audio File Serving:**
- Audio player loads from /api/audio/[filepath]
- Files served from /home/services/openotter/uploads/
- Security: Validate filepath is within upload directory before streaming

**Performance Optimization:**
- Memoize current utterance calculation in SyncedTranscript (useMemo)
- Throttle timeupdate event handling (100ms minimum)
- Consider virtualization for transcripts with 100+ utterances (deferred if not needed)

**Browser Compatibility:**
- HTML5 audio API supported in all modern browsers
- Test on Safari iOS (project target platform)
- Fallback for unsupported formats: show error message

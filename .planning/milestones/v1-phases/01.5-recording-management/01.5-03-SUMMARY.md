## What Was Built

### RecordingCard (`src/components/RecordingCard.tsx`)
Client component rendered as a Next.js Link. Displays filename (truncated), formatted date from SQLite datetime string, audio duration in MM:SS, speaker count, and a color-coded status badge (green=completed, yellow=processing, red=error). Active state via `active:bg-gray-100` for touch feedback.

### RecordingList (`src/components/RecordingList.tsx`)
Client component that maps recordings to RecordingCard components. Empty state shows a microphone SVG icon with a call-to-action message.

### AudioPlayer (`src/components/AudioPlayer.tsx`)
Client component using `forwardRef` and `useImperativeHandle` to expose a `seekTo(ms)` function to parent components. Renders an HTML5 `<audio>` element with native controls and a playback rate selector (1x, 1.25x, 1.5x, 2x). Accepts `/api/audio/{id}` proxy URL as `audioUrl` prop. The `onTimeUpdate` callback reports current time in milliseconds.

### SyncedTranscript (`src/components/SyncedTranscript.tsx`)
Client component composing AudioPlayer and an utterance list. Uses a `useRef<AudioPlayerRef>` to call `seekTo()` when users click an utterance. Throttles `timeupdate` to 100ms minimum and uses `useMemo` to compute the currently active utterance. Active utterance is highlighted with an indigo left border and background. Clicking an utterance seeks audio and auto-plays.

### Recordings List Page (`src/app/recordings/page.tsx`)
Server component that fetches all recordings from `/api/recordings` with `cache: 'no-store'`. Shows a user-friendly red error banner on failure. Passes recordings array to RecordingList.

### Recording Detail Page (`src/app/recordings/[id]/page.tsx`)
Server component using the Next.js 15 async `params` pattern (`await params`). Fetches from `/api/recordings/[id]`. Calls `notFound()` for 404s. Handles error and empty-transcript states. Constructs proxy URL `const audioUrl = \`/api/audio/${id}\`` and passes it to SyncedTranscript.

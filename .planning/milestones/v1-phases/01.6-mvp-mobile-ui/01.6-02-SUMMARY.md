# Phase 01.6 Plan 02: React Components Summary

**One-liner:** Eight React components built/restyled — ThemeProvider, StickyHeader with hydration-safe DarkModeToggle, RecordingCard/Skeleton/List/EmptyState using shadcn primitives, iOS-safe ExportButton, and SyncedTranscript restyled with shadcn tokens.

## What Was Built

This plan delivers the complete component layer for the Phase 1.6 redesign. All Wave 3 pages import from this set.

**New primitives:**
- `ThemeProvider` — thin wrapper around `next-themes` `NextThemesProvider`, ready to mount in the root layout
- `StickyHeader` — `sticky top-0 z-50` header with app name linking to `/`, `DarkModeToggle` sub-component (mounted guard prevents hydration mismatch), and "New Transcript" CTA linking to `/upload`
- `RecordingCardSkeleton` — mirrors `RecordingCard` layout using shadcn `Skeleton` components (title + meta + badge placeholder)
- `EmptyState` — Mic icon, locked copy ("No transcripts yet" / "Upload an audio file to get started."), "New Transcript" Button CTA
- `ExportButton` — synchronous `buildMarkdownExport` builds markdown before any `await`; clipboard path for ≤50KB; `.md` download via Blob URL for larger; sonner toast feedback

**Rewrites:**
- `RecordingCard` — replaced hand-rolled gray div with shadcn `Card`+`CardContent`; status badge uses shadcn `Badge` with `default`/`destructive`/`secondary` variants; date display uses `recorded_at` with `file_created_at` fallback (per D-10); filename strips extension (per D-12); imports `formatDuration`/`formatDate` from `@/lib/utils`; no `"use client"` (pure rendering)
- `RecordingList` — empty-state block removed; pure list renderer delegating to `RecordingCard`
- `SyncedTranscript` — `bg-indigo-50`/`bg-gray-50` replaced with `bg-accent`/`bg-muted`; active utterance uses `border-primary`; speaker labels replaced with shadcn `Badge variant="outline"` with inline hex color style; `cn()` used for conditional className; all audio sync logic (handleTimeUpdate, handleClickUtterance, useMemo throttle, AudioPlayerRef) preserved exactly

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] node_modules absent from worktree — installed before tsc verification**
- **Found during:** Task 1 verification
- **Issue:** The worktree had no `node_modules` directory (wave 1 agent installed packages in its own separate worktree). `tsc --noEmit` failed to find the TypeScript binary.
- **Fix:** Ran `npm install --include=dev` in the worktree to populate `node_modules` with all deps from `package.json`. TypeScript compilation then passed immediately.
- **Files modified:** `node_modules/` (not tracked in git — correct)
- **Commit:** Not committed (node_modules is gitignored)

**2. [Rule 1 - Bug] getSpeakerColor returns hex colors instead of Tailwind class strings**
- **Found during:** Task 2 implementation
- **Issue:** The plan specified `Badge variant="outline"` with `style={{ backgroundColor: speakerColor, color: '#fff' }}`, but the existing `getSpeakerColor` function returned Tailwind class strings (e.g. `"bg-indigo-100 text-indigo-800"`) which cannot be used as inline style values.
- **Fix:** Rewrote `getSpeakerColor` to return hex color strings (e.g. `"#6366f1"`) matching the same color palette, usable directly as `style.backgroundColor`. The `borderColor` was also set to the same hex for a clean Badge appearance.
- **Files modified:** `src/components/SyncedTranscript.tsx`
- **Commit:** d6be78f

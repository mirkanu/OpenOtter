## Recording Date From File

| Option | Description | Selected |
|--------|-------------|----------|
| Read m4a/mp3 metadata at upload time | Extract original recording date from audio tags via music-metadata | ✓ |
| Use upload date as 'recorded' date | Label as 'Uploaded', no extra library | |
| Show only transcription date | Just show transcribed date | |

**User's choice:** Read audio file metadata for recorded_at.

---

**Fallback clarification:** User asked whether "file created date" could be used as a fallback. After discussion, confirmed that `file.lastModified` from the browser File API gives the original device file date (set by iPhone at recording time), which is more accurate than server-side birthtime. User confirmed: store three dates — `recorded_at` (from audio metadata), `file_created_at` (from `file.lastModified`), `transcribed_at` (when transcription completed).

Display fallback: show `recorded_at` if available, else `file_created_at` with note "(file date)".

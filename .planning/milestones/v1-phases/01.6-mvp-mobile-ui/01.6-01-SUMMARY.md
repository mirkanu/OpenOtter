# Phase 01.6 Plan 01: Design System + Data Layer Foundation Summary

**One-liner:** shadcn/ui Neutral theme initialised with five components, CSS variables, and formatDuration/formatDate helpers; SQLite data layer extended with three date fields via idempotent migration.

## What Was Built

This plan establishes two parallel foundations required by all Wave 2 and Wave 3 plans:

1. **shadcn/ui design system** — `components.json` configured with Neutral base color and CSS variables. `tailwind.config.ts` created with `tailwindcss-animate` plugin and full shadcn token set. `globals.css` rewritten with light/dark CSS variable blocks. Five shadcn components installed: card, button, badge, skeleton, sonner. `src/lib/utils.ts` created with `cn()`, `formatDuration()`, and `formatDate()` helpers.

2. **Data layer extension** — `RecordingMetadata` interface gains three optional date fields (`file_created_at`, `recorded_at`, `transcribed_at`). `database.ts` gets an idempotent `runMigrations()` function that uses `PRAGMA table_info` to check for existing columns before issuing `ALTER TABLE ADD COLUMN`. `recording-store.ts` updated: `saveRecording()` now accepts `fileCreatedAt` and `recordedAt` parameters; `updateTranscribedAt()` added as a new export; both `getAllRecordings` and `getRecordingById` SELECT and return all three new fields.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] shadcn CLI 4.11.0 uses interactive wizard, not `--base-color` flag**
- **Found during:** Task 1
- **Issue:** `npx shadcn@latest init --base-color neutral` failed with "unknown option". CLI 4.11.0 dropped the flag-based init in favour of an interactive TUI wizard that cannot be passed flags for colour/style selection.
- **Fix:** Created `components.json` and `tailwind.config.ts` manually with the correct Neutral theme config, then ran `npx shadcn@latest add card button badge skeleton sonner` (non-interactive — only the `add` subcommand is non-interactive). The output is identical to what `init` would have produced.
- **Files modified:** `components.json`, `tailwind.config.ts`, `src/app/globals.css`
- **Commit:** e843b3d

**2. [Rule 3 - Blocking] Worktree node_modules missing devDependencies (tsc, @types/node)**
- **Found during:** Task 1 verification
- **Issue:** The worktree's `node_modules` only had ~86 packages; TypeScript, `@types/node`, and `@types/better-sqlite3` were absent, blocking `tsc --noEmit`.
- **Fix:** Added all missing packages to the worktree's `package.json` (`tailwindcss-animate`, `lucide-react`, `music-metadata`, `clsx`, `tailwind-merge`, `class-variance-authority`) and ran `npm install --include=dev` to populate the full devDependencies set.
- **Files modified:** `package.json`, `package-lock.json`
- **Commit:** e843b3d

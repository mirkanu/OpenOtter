# Phase 01.6 Plan 03: Date Pipeline — API Routes + AudioUploader Summary

**One-liner:** Upload route now captures fileLastModified (with parseInt+NaN guard) and music-metadata recorded_at (silent failure), forwarding both through the FormData → response → poll URL chain to saveRecording; transcribe route calls updateTranscribedAt on completion.

## What Was Built

This plan wires the three date fields introduced by Plan 01 to actual data sources across the upload-to-transcription pipeline.

**Upload route (`src/app/api/upload/route.ts`):**
- Reads `fileLastModified` from FormData (sent by AudioUploader)
- Validates with `parseInt(x, 10)` + NaN check; falls back to `new Date().toISOString()` if invalid (threat T-01.6-04)
- Calls `parseBuffer(buffer)` from music-metadata to extract `common.date` as `recordedAt`; wraps in try/catch — failure sets `recordedAt = null` and upload proceeds (threat T-01.6-06)
- Returns `fileCreatedAt` and `recordedAt` in the upload response JSON

**Transcribe route (`src/app/api/transcribe/[jobId]/route.ts`):**
- Imports `updateTranscribedAt` from `@/lib/recording-store`
- Reads `fileCreatedAt` and `recordedAt` from URL search params
- Passes both to `saveRecording()` alongside the existing `filename` and `filepath`
- Calls `updateTranscribedAt(recordingId, new Date().toISOString())` only when `result.status === "completed"` (not on error)

**AudioUploader (`src/components/AudioUploader.tsx`):**
- Appends `file.lastModified.toString()` to FormData as `"fileLastModified"` field
- Extended `onUploadComplete` callback signature: now passes `fileCreatedAt` and `recordedAt` to parent

**Parent pages (deviation — needed to complete the pipeline):**
- `src/app/upload/page.tsx`: updated `handleUploadComplete` to accept and forward `fileCreatedAt`/`recordedAt` as query params when routing to `/transcript/[jobId]`
- `src/app/transcript/[jobId]/page.tsx`: reads `fileCreatedAt`/`recordedAt` from search params; includes them in the transcription poll URL via `URLSearchParams`

**Types:**
- `UploadResponse` interface extended with `fileCreatedAt?: string` and `recordedAt?: string | null`

**next.config.js:**
- Added `serverExternalPackages: ['music-metadata']` to resolve ESM/CJS import compatibility

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical Functionality] Parent pages needed updating to complete the date pipeline**
- **Found during:** Task 2
- **Issue:** The plan specified updating `AudioUploader` to pass `fileCreatedAt`/`recordedAt` to the `onUploadComplete` callback, but the parent pages (`upload/page.tsx`, `transcript/[jobId]/page.tsx`) needed corresponding updates to forward those values through the route chain to the transcription poll URL. Without updating the parents, the date fields would be received by AudioUploader but silently dropped before reaching the transcribe route.
- **Fix:** Extended `handleUploadComplete` in `upload/page.tsx` to accept and forward the new date fields as URL params. Updated `transcript/[jobId]/page.tsx` to read `fileCreatedAt`/`recordedAt` from search params and include them in the poll URL via `URLSearchParams`.
- **Files modified:** `src/app/upload/page.tsx`, `src/app/transcript/[jobId]/page.tsx`
- **Commit:** ec9743b

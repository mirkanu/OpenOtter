The upload route currently does NOT call saveRecording — it only saves the file to disk and returns `{ filepath, filename }`. The transcribe polling route calls saveRecording later, receiving filename/filepath as query params.

To thread fileCreatedAt and recordedAt through to saveRecording in the transcribe route, the cleanest approach is to include them in the upload response, so the client can pass them as additional query params when polling.

**Changes to src/app/api/upload/route.ts:**

1. Add imports at top:
```typescript
import { parseBuffer } from 'music-metadata';
```

2. After `const buffer = Buffer.from(arrayBuffer);` (and before saveUploadedFile), add:

```typescript
    // Read fileLastModified from FormData (sent by AudioUploader per D-11)
    const fileLastModifiedStr = formData.get("fileLastModified") as string | null;
    const parsedMs = fileLastModifiedStr ? parseInt(fileLastModifiedStr, 10) : NaN;
    const fileCreatedAt = !isNaN(parsedMs)
      ? new Date(parsedMs).toISOString()
      : new Date().toISOString();

    // Extract recorded_at from audio file metadata — nullable, never blocks upload
    let recordedAt: string | null = null;
    try {
      const metadata = await parseBuffer(buffer, { mimeType: file.type });
      // common.date is "YYYY-MM-DD" string for M4A files (iPhone Voice Memos may or may not set it)
      recordedAt = metadata.common.date ?? null;
    } catch {
      // music-metadata failure must never block the upload
      recordedAt = null;
    }
```

3. Update the response to include the new date fields:
```typescript
    const response = {
      filepath,
      filename: file.name,
      fileCreatedAt,
      recordedAt,
    };

    return NextResponse.json(response, { status: 200 });
```

4. Update the UploadResponse type reference: since we're adding fields to the response that aren't in the existing UploadResponse interface, either update the interface in types.ts or use `as any` for this response. Preferred: update `src/lib/types.ts` UploadResponse to add the two optional fields:

```typescript
export interface UploadResponse {
  filepath: string;
  filename: string;
  fileCreatedAt?: string;
  recordedAt?: string | null;
}
```

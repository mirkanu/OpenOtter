  <context>
This is the only remaining work for the MVP. Cloudflare Access for a personal single-user app cannot be created via CLI in this workflow — it requires the Cloudflare Zero Trust dashboard. The steps below are exact; follow them in order.
  </context>
  <action>
Follow these exact dashboard steps:

1. Go to the Cloudflare Zero Trust dashboard: https://one.dash.cloudflare.com/
   - If prompted to select an account, choose the account that owns the `gsdlabs.dev` zone.

2. In the left sidebar, navigate to: **Access -> Applications**.

3. Click **Add an application**.

4. Select application type: **Self-hosted**.

5. On the application configuration screen, enter:
   - **Application name:** `OpenOtter`
   - **Session Duration:** `24 hours` (acceptable for personal use; longer = fewer re-logins)
   - **Application domain:**
     - Subdomain: `otter`
     - Domain: `gsdlabs.dev`
     - Path: leave blank (protects the entire app)
   - Leave other fields at their defaults.
   - Click **Next**.

6. On the **Policies** step, add a policy:
   - **Policy name:** `Allow Manuel`
   - **Action:** `Allow`
   - **Configure rules -> Include:**
     - Selector: `Emails`
     - Value: `manuelkuhs@gmail.com`
   - Click **Next**.

7. On the **Setup / Login methods** step:
   - Ensure at least one identity provider is enabled that can verify the email. The default **One-time PIN** method is sufficient (Cloudflare emails a code to manuelkuhs@gmail.com at login). Leave it enabled.
   - Click **Next** / **Add application** to save.

8. Confirm the application now appears in **Access -> Applications** with the domain `otter.gsdlabs.dev` and the `Allow Manuel` policy attached.

Note: No code changes, no tunnel changes, no PM2 restart. This is dashboard-only.
